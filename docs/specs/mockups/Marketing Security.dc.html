No raw data ever reaches an LLM

Models see schemas, governed definitions, and aggregate result shapes — never row-level data. Answer generation runs on statistical summaries computed inside your warehouse boundary.

Read-only warehouse access

Connections use a dedicated read-only role you control. AnalytIQ writes gold tables only to its own isolated schema. Nothing in your source data is ever mutated.

Deterministic validation gates

Generated SQL must pass safety analysis (read-only, row limits, time filters); results must pass shape and contract checks. Failures repair or halt — nothing unvalidated ships to a dashboard.

PII detected, masked, human-reviewed

Pattern and semantic detection flags likely PII on ingestion. Flagged columns are masked by default and only unmasked by an explicit steward decision — recorded in the audit log.

Every action in the audit log

Builds, approvals, shares, exports, permission changes, secret rotations — queryable, filterable, exportable as CSV/JSON, and streamable to your SIEM on Enterprise.

Row-level security with a simulator

Policies scope every query — chat, dashboards, embeds, exports. Admins preview any dashboard exactly as a given user would see it before granting access.

Signed, expiring share & embed tokens

Every external surface is a scoped signed token: expiration, password, domain allow-list, and permission flags — revocable instantly, individually or all at once.

Workspace-scoped everything

Artifacts, gold tables, models, and tokens live inside one workspace boundary. Cross-workspace access is off by default and always explicit.
